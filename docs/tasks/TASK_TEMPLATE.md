# タスクテンプレート

> このテンプレートを複製して、`docs/tasks/` 配下にタスクを作成してください。

## メタ
- 状態: `TODO | DOING | BLOCKED | REVIEW | DONE`
- 優先度: `High | Medium | Low`
- 担当: `@owner`
- 期限: `YYYY-MM-DD`（任意）
- 関連: Issue/PR/仕様（相対パスでリンク）

## 背景・目的
- なぜ必要か、成功条件は何かを1–3行で。

## スコープ（入/外）
- 入: 
- 外: 

## 完了の定義（DoD）
- 仕様と OpenAPI を更新し、`pnpm generate:types` 済み
- 型チェック/リンタ/テストがCIで通過（`pnpm typecheck`/`pnpm lint`/`pnpm test`）
- エラーフォーマットは `{ error:{code,message,details?}, requestId }` に準拠
- タイムゾーンは Asia/Tokyo、金額は円整数・税込、丸めは half-up
- ドキュメント（`docs/*`）と使用例（必要なら）を更新

## 作業項目（チェックリスト）
- [ ] 設計の明確化（入出力、境界条件、失敗時挙動）
- [ ] 実装（コード/設定）
- [ ] テスト（単体/統合/E2E/性能必要分）
- [ ] ドキュメント更新
- [ ] レビュー/リファクタ

## テスト計画
- 単体: 入力検証・境界値・例外系
- 統合: ルータ・DB・冪等・429/500
- E2E: オフライン→再送、PWA回り（必要時）
- 性能: 主要パスのP95（目標がある場合）

## リスク・依存
- 依存モジュール、マイグレーション、フラグ、運用前提

## ロールバック/フォールバック
- 失敗時の無効化手順・代替動作

## 参考
- 関連ファイル: `path/to/file.ts`
- 仕様: `requirements.md` / `openapi/openapi.yaml`
